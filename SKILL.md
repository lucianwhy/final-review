# 期末复习助手

帮助大学生在期末周快速完成复习。

**核心理念**：学生自带资料 → AI 解析生成题库 → 对话式刷题 → 错题专攻 → 知识点沉淀为 Markdown 知识库。

## 使用流程

### Phase 1: 资料上传
学生上传复习资料，支持格式：
- `.pptx` — PowerPoint 幻灯片
- `.pdf` — PDF 文档
- `.doc/.docx` — Word 文档
- `.png/.jpg/.jpeg/.bmp/.gif/.webp` — 图片（OCR 识别）
- `.txt/.md` — 纯文本/Markdown

可一次上传多个文件，自动合并为一个科目的复习资料。

### Phase 2: 内容解析
调用 `python scripts/parse_file.py <文件路径>` 提取文本内容。

### Phase 3: 题库生成
AI 基于解析后的文本，生成结构化题库（JSON 格式）。要求：
- 按知识点分类出题
- 题型：单选、判断、填空、简答
- 每道题包含：题目、选项（如适用）、正确答案、详细解析、所属知识点

题库保存到 `storage/<科目名>/quiz.json`。

### Phase 4: 知识库生成
AI 基于解析后的文本，生成 Markdown 知识点总结。要求：
- 按章节/主题组织
- 每个知识点包含：概念定义、核心要点、易错点、相关公式/定理
- 保存到 `storage/<科目名>/knowledge_base/index.md`
- 刷题过程中答错的知识点，自动补充标注到知识库

### Phase 5: 刷题模式
调用 `python scripts/review.py <科目名>` 进入交互式刷题。

交互流程：
1. AI 依次展示题目
2. 学生回答
3. AI 判断对错，给出详细讲解
4. 错题自动记录到 `wrong_answers.json`
5. 支持中途退出，下次续刷

### Phase 6: 薄弱点专攻
全部刷完后，AI 分析错题本：
- 按知识点统计正确率
- 正确率低于 60% 的知识点列为薄弱点
- 针对薄弱点追加生成新题进行加练

## 知识库 Markdown 格式

```markdown
# 数据结构 - 知识点总结

## 1. 线性表

### 1.1 数组
- **定义**：连续内存空间存储的同类型数据元素集合
- **核心要点**：
  - 随机访问 O(1)
  - 插入/删除 O(n)（需要移动元素）
  - 适用于读多写少的场景
- **易错点**：
  - 数组下标从 0 开始
  - 动态数组扩容时的时间复杂度均摊分析

### 1.2 链表
- **定义**：通过指针连接的非连续存储结构
- **核心要点**：
  - 查找 O(n)
  - 插入/删除 O(1)（已知前驱节点时）
  - 单向链表、双向链表、循环链表的区别
- **易错点**：
  - 操作链表时注意空指针和头节点处理
  - 面试常考：反转链表、合并有序链表、判断环

## 2. 树
...
```

## 脚本说明

| 脚本 | 功能 |
|------|------|
| `parse_file.py <文件路径>` | 解析文件，输出提取的文本 |
| `review.py <科目名> [mode]` | 刷题模式。mode: `all`(全部), `wrong`(错题), `weak`(薄弱点) |
| `utils.py` | 数据存储/读取工具函数 |

## 存储结构

```
storage/
└── <科目名>/
    ├── parsed_content.md      # 解析后的原文
    ├── quiz.json              # 生成的题库
    ├── wrong_answers.json     # 错题本
    ├── progress.json          # 刷题进度
    └── knowledge_base/
        └── index.md           # 知识点总结
```
