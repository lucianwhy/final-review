## 期末复习工作流

### 目标
AI 任务不是泛化讲解。目标是：
1. 按真实考试题型和高频考法出题。
2. 按题源权重优先复用最可能上考场材料。
3. 输出可直接复习、可直接背、可直接拿分内容。
4. 解析不只讲对错，还要讲考试怎么得分。

### 题源优先级
出题时按以下优先级取材，不可平均分配：

1. 历年真题：最高优先级。
2. 老师 PPT：高优先级。
3. 平时作业 / 阅读作业：中高优先级。
4. 速成课题目：较低优先级。
5. AI 自补题：最低优先级，仅在材料不足时使用。

### 速成课使用规则
速成课题目优先级低，但知识点梳理价值高，主要用于：
1. 提炼核心知识点。
2. 搭复习框架和结构。
3. 总结重点、易错点、常考点。
4. 辅助判断哪些内容更值得重点复习。

### 材料处理规则
1. 所有上传材料先统一转成 Markdown。
2. 默认建议使用 `markitdown` 工具转换。
3. 转完先清洗，再提知识点、再出题。
4. 清洗重点：去乱码、去重复、去无关废话，保留核心定义、性质、结论、典型考法、易错点。

原则：复习材料以“清晰、可背、可考”为第一目标，不以“完整存档”为第一目标。

### 出题规则
1. 先明确本课会考哪些题型，再按题型出题。
2. 每个知识点按密度和重要度出 `1~6` 题，不可机械平均。
3. 知识点越密、越高频、考法越多，题量越多。
4. 题型要匹配知识点最可能考法，不可乱配。

### 题源标记规则
每道题建议做轻量类别级题源标记，例如：
- `题源：历年真题`
- `题源：老师PPT`
- `题源：平时作业`
- `题源：速成课`
- `题源：综合改编（历年真题 + PPT）`
- `题源：AI补充`

默认不要求精确追踪到文件级、页码级，除非用户明确要求。

### 输出格式规则
普通正文输出：
1. 前面只放题目。
2. 所有答案统一放最后。
3. 题目区和答案区明确分开。

HTML 输出：
1. 答案放在题目下面。
2. 答案默认折叠隐藏。
3. 点击后再展开答案和解析。

### 解析规则
每道题解析尽量包含：
1. 正确答案 / 参考答案。
2. 本题考哪个知识点。
3. 这类题常见考法。
4. 考试时如何尽可能拿分。

客观题重点补：
1. `这题核心知识点`
2. 关键定义、性质、判定依据
3. 易混点

主观题重点补：
1. `这题答题核心点`
2. `必须出现`
3. `常见失分点`
4. 标准作答框架

### 得分导向规则
解析必须服务考试得分，不写空话。优先补：
1. 看到这类题先写什么。
2. 哪些关键词必须出现。
3. 哪些步骤能抢步骤分。
4. 哪些错误最容易丢分。

### 默认流程
1. 先明确考试题型和常见考法。
2. 用 `markitdown` 把全部材料转成 Markdown。
3. 清洗 Markdown，去噪提核心。
4. 按真题、PPT、作业、速成课建立知识点池。
5. 按题源优先级和知识点重要度出题。
6. 每个知识点生成 `1~6` 题。
7. 每题标记轻量题源类别。
8. 题答分离。
9. 客观题补“这题核心知识点”。
10. 主观题补“这题答题核心点”“必须出现”“常见失分点”。
11. 解析顺手补考试技巧和得分点。
